Nginx IP透传

最近在折腾家里的nas，因为家里存在多个nginx服务器，如果不能正确的配置nginx，一些依赖于客户端IP来判断内外网环境的服务（比如jellyfin）就会出错。

经过查阅资料发现，nginx存在nginx_http_realip_module模块，可以很好的解决多级代理下IP透传的问题。

假设内网存在三个nginx服务器：

  • 172.19.0.8
  • 172.19.0.9
  • 172.19.0.10

其中，172.19.0.10上部署了后端应用。则配置如下：

# 172.19.0.8

server {
    
    ...
    
    location / {
        proxy_pass http://172.19.0.9;

        ...

        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        ...
    }
}

# 172.19.0.10

server {

    ...

    #定义可信任的代理服务器地址
    set_real_ip_from 172.19.0.8;
    set_real_ip_from 172.19.0.9;

    #指定从哪个HTTP报头里检索IP信息
    real_ip_header X-Forwarded-For;

    #递归排除每个代理服务器的IP
    real_ip_recursive on;
    
    ...
}
